Introduction to SQL

Overview

What is SQL?

SQL (Structured Query Language) is a standard language for managing and manipulating databases. It is used to perform tasks such as querying data, updating records, and managing database structures.

Basic SQL Queries

Fundamentals

SELECT Statement

-- Retrieve all columns from the 'employees' table
SELECT * FROM employees;

-- Retrieve specific columns from the 'employees' table
SELECT first_name, last_name, salary FROM employees;

Advanced SQL Queries

Complex Queries

JOIN Operations

-- Retrieve orders and their corresponding customer information
SELECT orders.order_id, customers.customer_name
FROM orders
INNER JOIN customers ON orders.customer_id = customers.customer_id;

Subqueries

-- Retrieve employees who earn more than the average salary
SELECT * FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

SQL Functions

Common Functions

Aggregate Functions

-- Count the number of employees in the company
SELECT COUNT(*) AS number_of_employees FROM employees;

String Functions

-- Combine first and last names into a full name
SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM employees;

SQL Data Types

Understanding Data Types

Numeric Data Types

-- Numeric types include INT, DECIMAL, FLOAT, etc.
CREATE TABLE products (
    product_id INT PRIMARY KEY,
    price DECIMAL(10, 2) NOT NULL
);

String Data Types

-- String types include CHAR, VARCHAR, TEXT, etc.
CREATE TABLE users (
    user_id INT PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL
);

Date and Time Data Types

-- Date and time types include DATE, DATETIME, TIMESTAMP, etc.
CREATE TABLE events (
    event_id INT PRIMARY KEY,
    event_date DATETIME NOT NULL
);

SQL Transactions

Managing Transactions

BEGIN, COMMIT, and ROLLBACK

-- Begin a transaction
BEGIN;

-- Perform SQL operations
UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;

-- Commit the transaction
COMMIT;

-- If there is an error, rollback the transaction
ROLLBACK;

SQL Views and Stored Procedures

Advanced SQL Features

Views

-- Create a view to simplify complex queries
CREATE VIEW employee_view AS
SELECT first_name, last_name, salary
FROM employees
WHERE department = 'Sales';

Stored Procedures

-- Create a stored procedure for repetitive tasks
CREATE PROCEDURE GetEmployeeById(IN emp_id INT)
BEGIN
    SELECT * FROM employees WHERE employee_id = emp_id;
END;
